# Журнал работ на строительном объекте

Мы строим внутренние инструменты для управления строительными проектами. Одна из повседневных задач — фиксация выполненных работ на объекте. Твоя задача — собрать простой «Журнал работ», в котором прораб может вести учёт того, что было сделано на объекте за каждый день.

## Функциональные требования

### Минимум (обязателен):
- Список записей журнала. Таблица или список, где видны: дата выполнения, наименование вида работ (например, «Кладка перегородок», «Монтаж опалубки»), объём с единицей измерения (например, «24 м³»), ФИО исполнителя (бригадира/работника). Должна быть возможность отсортировать или отфильтровать записи по дате.
- Добавление записи. Форма для создания новой записи с валидацией обязательных полей. Данные сохраняются в базу данных.
- Удаление записи.

### По желанию (для тех, кто хочет показать больше):
- Редактирование существующей записи.
- Простейший справочник видов работ: вместо свободного ввода — выбор из предзаполненного списка (можно захардкодить или сделать отдельную таблицу в БД).

## Технические требования

### Обязательно:
- Фронтенд — React + TypeScript
- Данные хранятся в базе данных (не в памяти, не в файле)
- Взаимодействие фронтенда с бэкендом через API

### На твой выбор: 
- Бэкенд
- База данных
- Остальной инструментарий (ORM, UI-библиотеки, стейт-менеджмент, средства сборки) — на твоё усмотрение
